Log in to Elja

  • Log in to Elja and navigate to /hpcdata/Mimir/yourusername.
  • Make a new directory for logfiles, and then within that make a new directory for today’s date. Navigate into this directory.
  • Type: screen -L. It will open a new screen for you and make a screenlog file that you can later download and view the text.
  • Compute to a node using a command like this: srun -N 2 -n 48 --pty /bin/bash. Do not run anything before entering a node, it will slow down the whole Elja system otherwise and impact everyone.
  • Now you can navigate into a folder with your files and run different programs.
  • I first like to have each set of files for each sample in a separate folder, so that I can process them all at once using folder names. I use the following shell script for this:
#!/bin/sh

#Script to make folders based on file names and move files into the folders

for f in *.fastq.gz
do
  subdir=${f%%_L*}
  [ ! -d "$subdir" ] && mkdir -- "$subdir"
  mv -- "$f" "$subdir"
done

Save this file as a .sh file, i.e. “move_folders.sh” and then upload to Elja. Before running you need to make the script executable by using the following command:

chmod u+x move_folders.sh

Then you can run the script like this:

./move_folders.sh
  • For quality checking you can run FastQC. I already trimmed by quality and trimmed adapter sequences during demultiplexing, so might not be necessary. If you want, here is the command to run fastQC on everything:
find . -maxdepth 1 -mindepth 1 -type d -exec bash -c "cd '{}' && pwd && fastqc *.fastq.gz" \;

Alignment

We can use bowtie2 for aligning to the relevant genome. You can download ready-made bowtie2 indexes from the Bowtie2 manual page or from Illumina iGenomes or you can build your own. For the E. coli reference genome you should download Escherichia coli strain K12, MG1655 from iGenomes. Refer to the Bowtie2 manual for use if you want to build your own index. To download the E.coli index directly to Elja, you can use the following command:

wget http://igenomes.illumina.com.s3-website-us-east-1.amazonaws.com/Escherichia_coli_K_12_MG1655/NCBI/2001-10-15/Escherichia_coli_K_12_MG1655_NCBI_2001-10-15.tar.gz

Now we can start alignment. I would recommend having a couple of screens running at the same time, one for aligning to the E.coli genome and one to your organism, so that you can save time. Make sure that you remember the screen -L command in a new folder so that you will get a saved text output of the alignments. For the E. coli genome I ran the following shell script. You can do something similar for your organism. You might need to change the names of the fastq files if they are different, i.e. from a different run. Also, make sure to change the name of the .sam file so that you will have different file names for aligned E. coli reads and those of your organism.

#!/bin/sh


#Script to run bowtie2 on everything
find . -maxdepth 1 -mindepth 1 -type d -exec sh -c "cd '{}' && pwd && bowtie2 -p 24 -x /hpcdata/Mimir/kimberl/iGenomes/Escherichia_coli_K_12_MG1655/NCBI/2001-10-15/Sequence/Bowtie2Index/genome -1 '{$1}'_L004_R1_001.fastq.gz -2 '{$1}'_L004_R2_001.fastq.gz -S '{$1}'_e_coli.sam" \;

Preparing bedGraph files for use in SEACR

We need to prepare bedGraph files that are normalised to the E.coli alignment. BedGraph files are the required input for SEACR and can later be used for visualisation. For this we need to make a shell script that reads in the csv file we prepared earlier and runs bedtools for each bam file. Remember to activate the bedtools module before starting.

#!/bin/sh


#Using awk to remove the quotation marks from the csv file
awk '{gsub(/"/,"")};1' E_coli_align.csv >  E_coli_align_clean.csv

INPUT=E_coli_align_clean.csv
OLDIFS=$IFS
IFS=','
[ ! -f $INPUT ] && { echo "$INPUT file not found"; exit 99; }


while read Sample read_number percent_align norm_factor Antibody
do
    echo "Name : $Sample"
    echo "Read number : $read_number"
    echo "Percentage alignment to E. coli : $percent_align"
    echo "Normalisation factor : $norm_factor"
    echo "Antibody : $Antibody"

  ls ${Sample}/

  bedtools genomecov -bg -pc -scale $norm_factor -ibam ${Sample}/${Sample}.bam > ${Sample}.norm.bedgraph


  echo -e "\n\n"

done < $INPUT
IFS=$OLDIFS

SEACR for calling peaks

Move normalised bedgraph files into separate folders based on which antibody was used, then run the following script. Here we are trying two methods, calling peaks with the “non” setting to use the E.coli normalisation that we already performed, and with the “norm” method to run normalisation within SEACR. After comparing both outputs in IGV, it seems there isn’t a huge difference, so for H3K36me3, I will continue with the “non” normalised bed files of peaks.

#!/bin/sh

declare -a H3K36_files
for file in H3K36/*.bedgraph
do
    H3K36_files=("${H3K36_files[@]}" "$file")
done

declare -a IgG_files
for file in IgG/*.bedgraph
do
  IgG_files=("${IgG_files[@]}" "$file")
done


# Using [*]
echo -e '\nUsing [@] -----------------------------------'
for i in "${!H3K36_files[@]}"
do
    echo -e "Sample: *** $i ***"
    printf "%s matches with %s\n" "${H3K36_files[i]}" "${IgG_files[i]}"

    bash SEACR_1.3.sh ${H3K36_files[i]} ${IgG_files[i]} non stringent ${H3K36_files[i]:0:27}

    bash SEACR_1.3.sh ${H3K36_files[i]} ${IgG_files[i]} norm stringent ${H3K36_files[i]:0:27}_norm

    echo -e "\n\n"


done
